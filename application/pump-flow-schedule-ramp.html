유량 스케줄은 '언제 얼마를 흘릴지'를 미리 시퀀스로 짜 두고 자동 실행하는 것입니다. 일정 시간 특정 유량 유지, 선형 증감(ramp), 단계별 스텝을 조합해 유량 프로파일을 만들고, 한 번 시작하면 사람이 없어도 설정대로 유량이 바뀝니다. 이 시퀀스를 레시피로 저장하면 다음 실험에서 그대로 불러 동일 조건을 재현할 수 있습니다.

1. 유량 스케줄이란 — 고정·ramp·스텝

수동 제어는 "지금 유량을 얼마로" 한 번 정하는 것이지만, 스케줄 제어는 시간축 위에 유량을 그리는 것입니다. 가장 기본은 세 가지 블록입니다. ① 고정 — 정해진 시간 동안 같은 유량 유지, ② ramp — 한 유량에서 다른 유량으로 서서히 전환, ③ 스텝 — 특정 시점에 유량을 즉시 다른 값으로 변경. 이 블록들을 이어 붙이면 실험 전체의 유량 프로파일이 됩니다.

2. ramp — 급변 없이 조건을 전환

ramp(선형 증감)는 유량을 정해진 시간에 걸쳐 서서히 바꿉니다. 급격한 유량 변화는 세포에 전단 스트레스를 주거나 회로에 압력 충격을 줄 수 있는데, ramp로 완만하게 올리고 내리면 이를 피할 수 있습니다. 예를 들어 관류를 시작할 때 0에서 목표 유량까지 10분간 ramp로 올리면, 배양물이 갑작스러운 흐름에 노출되지 않습니다.

3. 레시피 저장·반복 — 재현의 핵심

한 번 짠 시퀀스를 레시피 파일로 저장했다가 다음 실험에서 그대로 불러 실행하면, 사람이 손으로 맞추는 편차가 사라집니다. "이 실험은 어떤 유량으로 돌렸나"가 파일 하나로 남아, 실험 간 조건이 동일하게 재현됩니다. 재현성이 성립 조건인 연구에서 특히 중요합니다.

이럴 때 씁니다

  • 관류 배지교환 스케줄 — 정해진 주기로 유량을 전환해 배지를 갈아줌
  • 농도 구배·조건 스윕 — 유량을 단계적으로 바꿔 여러 조건을 한 번에 탐색
  • 장시간 무인 프로토콜 — 밤사이 유량 변화를 사람 없이 자동 진행

Python 예제 — Python으로 펌프 유량을 시간에 따라 바꾸려면?

유량 setpoint 레지스터에 값을 순서대로 쓰면 됩니다. 유지 → 증감(ramp) → 스텝을 리스트로 만들고 각 구간마다 write 후 대기하면, 사람이 없어도 시간표대로 유량이 바뀝니다. (정확한 레지스터 주소·명령·바이트오더는 각 장비 매뉴얼을 따르세요.)

import minimalmodbus, time
# 레지스터 주소·바이트오더는 장비 매뉴얼 참조
REG_FLOW, REG_RUN = 0x0000, 0x0000   # ← 매뉴얼 값으로 대체

pump = minimalmodbus.Instrument('COM3', 1)          # 포트, 슬레이브 주소
pump.serial.baudrate = 9600
pump.serial.parity = minimalmodbus.serial.PARITY_EVEN   # 예: 8E1

pump.write_register(REG_RUN, 1, functioncode=6)     # Start
for flow in [2.0, 5.0, 8.0, 12.5]:                  # 유량 setpoint 시퀀스(mL/min)
    pump.write_float(REG_FLOW, flow)                # 유량 변경
    time.sleep(600)                                 # 각 단계 10분 유지
pump.write_register(REG_RUN, 0, functioncode=6)     # Stop
