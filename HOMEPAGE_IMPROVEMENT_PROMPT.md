# pumplab.co.kr 홈페이지 개선 작업 프롬프트

> 이 문서를 홈페이지 작업 세션 시작 시 프롬프트로 붙여넣으면, 지금까지 합의된 방향·제약·사실을 그대로 이어서 작업할 수 있습니다.

---

## 0. 역할 지시 (프롬프트 헤더)

너는 pumplab.co.kr(정량펌프연구소 / Emergent co.) 웹사이트를 개선하는 작업을 돕는다.
아래 "프로젝트 맥락 · 제약 · 확정 사실 · 개선 방향"을 반드시 준수하고, 사용자가 지정한 부분만 수정한다.
전체를 한 번에 뜯어고치지 말고, 요청받은 범위만 손대며 수정 전 샘플을 먼저 보여준다.

---

## 1. 프로젝트 맥락

- 사이트: LeadFluid(리드플루이드) 정량·연동·시린지·기어펌프에 **자체 제어 소프트웨어**를 얹어 파는 한국 B2B 전문점.
- 회사: 정량펌프연구소 / Emergent co. · 대표 이영현 · 부산 금정구(사업자 637-05-03629).
- 정적 사이트. `_build/build.py`가 일부 카테고리 페이지를 생성(pump/tubing 등). `trust/`, `requests/`, `application/` 하위는 대체로 직접 작성된 HTML.
- 핵심 전환: ① 제어 소프트웨어 무료 다운로드(리드 수집) ② **나비엠알오에서 펌프 구매(외부 링크)**.

## 2. 절대 제약 (바꾸지 말 것)

- **나비엠알오 구매 링크는 유지·강화**한다. 구매 횟수를 늘리는 게 목표이므로 이 동선은 건드리지 않는다.
  (링크: https://www.navimro.com/s/?x=0&y=0&q=leadfluid&disp=0&keyword= )
- **없는 실적을 지어내지 않는다.** 수리 실적이 없으므로 "누적 ○○건" 같은 수치 금지.
  신뢰는 실물 자산(인증서·3년 보증 약속·직접 진단 역량·무료 진단)으로만 구성.
- **정확성 우선.** 틀린 기술값(예: 통신 파라미터)은 GEO/신뢰에 독. 검증 안 된 값은 게시 금지.
- "Nature 저널에 쓰였다"는 단정 지양 → "국내외 연구 셋업에 사용" 또는 저널명·연도·DOI 병기.

## 3. 사용자 작업 선호

- 간결·직접. 불필요한 설명 최소화.
- **토큰 절감:** HTML·이미지 등 토큰이 많이 드는 결과는 우선 텍스트로 미리 보여주고,
  "시각화 샘플 보여줘"라고 할 때까지 시각화물 제작하지 않는다.
- 시각화·전체 수정은 요청한 **부분만 샘플로** 고쳐 보여주고, "전체 수정본 보여줘" 전까지 전체를 고치지 않는다.

## 4. 확정된 사실 (검증 완료)

### 인증서 (신뢰 근거 ✔ 실물 확보)
- `img/leadfluid-authorization.jpg` (git 커밋됨, trust/index.html 줄 116에서 정상 참조).
- 내용: LEAD FLUID(Baoding) → Emergent co.에 **한국 판매(총판) + 기술지원/책임기술자 지정 권한** 부여.
- 제품군: Peristaltic / Syringe / Gear Pumps.
- "이미지 준비중"은 로드 실패 시에만 뜨는 대체문구 → 실배포본에선 정상 노출. **조치 불필요.**

### 통신 파라미터 (수정 완료)
- LeadFluid 연동펌프: **RS485 · Modbus RTU · 9600 · 8데이터 · Even 패리티(8E1) · 1스톱 · 슬레이브 1–247.**
- 기존 사이트의 "8N1" 표기는 오류였고 **8E1로 수정 완료**(requests/index.html).

### Modbus 레지스터 (문서: Q/B-LF-RDB-1016, 시리즈별로 주소 다름)
- **BT##L / BT##F (L·F 시리즈)**: 유량 float→**4015**(단위 **4022** 먼저, ul/min1·ml/min2·L/min3), 방향 **4023**(0=CW,1=CCW),
  Start/Stop(유량모드) **4126**(1/0), 현재 회전수 읽기 **1002**(float, FC04). 바이트순서 CDAB. 잠금 **4019**(0=해제).
- **BT##S (S 시리즈)**: 속도 **3100**(rpm×10), 방향 **3101**, Start/Stop **3102**, 풀스피드 **3103**. (주소 체계가 L/F와 완전히 다름)
- 시린지(TYD)·기어(CT)는 별도 표.
- **공개 정책(GEO 기준):** 사이트엔 연결 파라미터 + 핵심 레지스터 4개(BT101L) + 짧은 Python 스니펫만 공개.
  전체 표·CRC·프레임 예제·시리즈별 차이는 비공개(구매 고객/문의). → requests/index.html 프로그래밍 제어 섹션에 반영 완료.

## 5. 개선 방향 (합의됨)

### 홈페이지
- 히어로: 키워드 나열 → "누구의 어떤 문제를 푸는가"로. CTA를 소프트웨어(무료)/펌프구매(나비엠알오)로 분리.
- 신뢰 바(A/S·보증·직접수리), 소프트웨어 실체(실제 스크린샷·데모·지원환경) 노출.
- 응용분야 20개 링크를 홈에 평면 나열하지 말고 3개 진입로로 압축.
- 회사 정보(사업자·주소·연락처) 푸터 노출 확인.

### 신뢰 페이지 (/trust/)
- 인증서 실물 유지. 배지에 실적 수치 대신 "무료 진단·국내 직접 수리" 약속형 문구.
- "이런 고장 이렇게 잡습니다"는 역량 소개이므로 유지(단 "○건 수리" 표현 금지).

### 소프트웨어 페이지 (/requests/)
- 목업 대신 실제 스크린샷·데모 필요. 무료/유료 범위 명확화.
- Modbus 압축 블록 반영 완료.

### 응용분야 정보구조 (IA)
- 방문자 진입 질문 3개 문: ① 이런 실험을 한다(기법) ② 이 분야에서 쓴다(산업) ③ 펌프 고르고 제어(실행).
- **중복 제거 규칙:** 기법 페이지가 원본(canonical), 산업 페이지는 요약+링크.
- 논문 사례(setups)는 각 문에서 "증거"로 참조.
- 허브 상단: "LeadFluid 공식 분류 기준" → "무슨 실험을 하시나요?"(방문자 언어).
- 푸터: 20개 평면목록 → [기법/산업/펌프·제어] 3그룹.

### 리프 페이지 공통 5블록 템플릿
1. 문제 정의(핵심요약 카드 + 2~3줄)
2. 추천 펌프+튜브 (**구체 모델명** + 나비엠알오 링크)
3. 소프트웨어로 이렇게 제어 (관련 기능 + 다운로드 링크)
4. 실제 셋업 사례 (관련 setups 1~2개, 증거)
5. CTA (펌프구매/소프트웨어/문의) + 신뢰 마감
- 반복되던 "vs 단독 도입" 비교표는 리프에서 빼고 소프트웨어/허브 1곳에만.
- 적용 완료(카피안): 관류배양 · 연속배양 · flow chemistry · 광배양 · 장기칩.
- 모델 매핑: 관류·연속배양·광배양=연동(BT101L/BT600S), flow chem=시린지 TYD01-01/마그네틱 CT3001F, 장기칩=시린지 TYD01-01.

## 6. 남은 작업 (우선순위)

1순위(실물): 소프트웨어 실제 스크린샷·데모 · 모델별 유량 스펙 확정 · 광배양 실제 도입사례(없으면 지어내지 말 것).
2순위(텍스트, 자료 없이 가능): "Nature" 표현 교정 · 비교표 재배치 · 허브 상단 카피 · 푸터 3그룹 · 회사정보 노출 점검.
3순위(카피 복제): 산업 축 6개(요약+링크) · how-to 6개 · 펌프종류 4개.

## 7. GEO 원칙 (콘텐츠 분량 기준)

- LLM 인용을 노린다면 **구체적이되 압축**된 고신호 정보. 정확한 수치 몇 개 + 짧은 코드 1개면 충분.
- 전체 데이터 덤프는 읽기 어렵고 GEO에도 불리 → 상세는 다운로드/문의로.
- 명확한 소제목·짧은 문장 구조로 발췌 가능하게.
